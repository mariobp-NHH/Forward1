What is AWS?

AWS (Amazon Web Services) is a comprehensive, evolving cloud computing platform provided by Amazon that includes a mixture of infrastructure-as-a-service (IaaS), platform-as-a-service (PaaS) and packaged-software-as-a-service (SaaS) offerings. AWS services can offer an organization tools such as compute power, database storage and content delivery services.

AWS offers more than over 200 fully featured services from data centers globally. Those services includes storage, databases, artificial intelligence, load balancing, scalability, etcetera. The versability and flexibility of AWS help to customize and scale up your project easily and economically. In the video in which I explain how to create users and groups, I explain some of the features used by AWS that we are going to use during the course. Create a user, a group and IAM

AWS Identity and Access Management (IAM) is a web service that helps you securely control access to AWS resources. With IAM, we can centrally manage permissions that control which AWS resources users can access. We use IAM to control who is authenticated (signed in) and authorized (has permissions) to use resources.

When we create an AWS account, you begin with one sign-in identity that has complete access to all AWS services and resources in the account. This identity is called the AWS account root user and is accessed by signing in with the email address and password that we used to create the account. AWS strongly recommend that you don't use the root user for your everyday tasks. Safeguard your root user credentials and use them to perform the tasks that only the root user can perform. That is the reason because create users and groups by using IAM is extremely useful.

In the YouTube video for this section, I explain how to create groups, users, and how to add users to a group. This will be very useful for our project, since each single member of the group can use AWS without compromising the security of the root account.

Deploying our app 1: AWS Elastic Beanstalk.

Elastic Beanstalk is a service for deploying and scaling web applications and services. Upload your code and Elastic Beanstalk automatically handles the deployment—from capacity provisioning, load balancing, and auto scaling to application health monitoring.

When we use Elastic Beanstalk, we create an environment in which we have access to many AWS services as EC2, S3, Simple Notification Service (SNS), CloudWatch, autoscaling, and Elastic Load Balancers. That feature of Elastic Beanstalk facilitates the deployment of our app and helps us to scale up our project easily and economically.

Deploying our app 2: AWS CodePipeline.

AWS CodePipeline is a fully managed continuous delivery service that helps you automate your release pipelines for fast and reliable application and infrastructure updates.

AWS CodePipeline helps us to connect our GitHub repository with our Elastic Beanstalk in a very efficient way. So, when we introduce some changes in our code and commit those changes in GitHub, we can deploy those changes easily in the server by using CodePipeline.

Weekly challenge

In this weekly challenge we focus on the use of branches since that will be a useful to cooperate developing the app in the next chapters. After introducing some changes in GitHub, we release those changes by using CodePipeline to upload our webpage in AWS.

You have to modify the login function in the package users to pass your name to the template login.html. When you click in the login link in the navigation bar or in the home page, you can see the next message: “Hello {{your name}}, you can start to use the carbon app.”

The objective of this weekly challenge is to help you to start to work with branches. In the weekly challenge, you have to explain to the rest of the class how you have created a branch, how you modify the code in your computer, and how you merge the branch that you created with the main branch. After that, you must use CodePipeline to release the changes introduced in GitHub. So, that you can upload your app in AWS.
